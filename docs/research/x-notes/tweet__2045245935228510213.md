## Tweet text

with Codex app automations, I have settled into is the harness engineering style of it: automations live in markdown files in docs/automations. The codex app automation prompt is then 2 sentences:
> you are automation X agent. the runbook is in <file>.

this makes automations:
* code reviewed
* repo owned rather than single app instance owned
* versioned

## Notes

- Encourages keeping automations as **repo-owned markdown runbooks** with minimal “launcher” prompts.
