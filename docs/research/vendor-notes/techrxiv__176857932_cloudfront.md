# TechRxiv — Agent Skills from the Perspective of Procedural Memory (survey)

## What it is

TechRxiv preprint (Jan 16, 2026) titled **“Agent Skills from the Perspective of Procedural Memory: A Survey”**.

## Key ideas worth copying

- Frames **agent skills as procedural memory**: reusable, temporally-extended behaviors bridging goals → actions.
- Describes a lifecycle:
  - **acquisition** (from interaction/experience)
  - **representation** (as modular, discoverable units with metadata)
  - **invocation** (selecting/composing skills during execution)
  - **refinement** (feedback-driven improvement)
- Emphasizes that skills can be:
  - explicit, stored instructions/workflows/scripts/tool specs
  - implicit, internalized policies/parameters

## How it maps to PlotLot

- Supports the “skills as repo artifacts” direction:
  - feasibility workflows should become explicit skills/runbooks
  - skills need metadata (when to use, inputs/outputs, risks)
  - skills should be testable + evaluable (holdouts, regressions)
