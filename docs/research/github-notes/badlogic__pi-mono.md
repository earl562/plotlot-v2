# badlogic/pi-mono

## What it is

- A large “agent toolkit” monorepo (CLI + libraries) for building agents.

## Notable primitives

- Modular packages called out in README:
  - unified multi-provider LLM API
  - agent runtime w/ tool calling + state management
  - coding agent CLI
- Emphasis on **sharing OSS sessions** (real-world traces) to improve agents.

## How it maps to PlotLot

- Design reference for:
  - keeping the harness split into libraries vs. product UIs
  - session/trace capture as a first-class artifact for eval + debugging
